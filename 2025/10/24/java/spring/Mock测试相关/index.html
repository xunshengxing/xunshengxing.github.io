Mock测试相关

发表于2025-10-24|更新于2025-10-24|spring
|总字数:1.3k|阅读时长:5分钟|浏览量:

测试

测试分为两类，

  • 单元测试（不依赖Spring容器，测试单个类或方法），使用JUnit + Mockito

  • 集成测试（测试多个组件，需要启动springboot）,@SpringBootTest、@WebMvcTest、MockMVC

Mockito

Mockito用于创建模拟对象，隔离其他环境来专注测试某个类

  1. Mock（完全模拟）：

    • 创建一个假的对象，用来替代真实对象。
    • 这个对象的行为可以由你来定义（比如调用某个方法时返回什么值）。
    • 目的是隔离依赖，避免在单元测试中真正去访问数据库、调用远程接口等。

  2. Stub（行为预设）：

    • 定义模拟对象在被调用时应该返回什么值或执行什么动作。
    • 例如：when(mockService.getData()).thenReturn("mocked data");
    • 这样，当测试代码调用 mockService.getData() 时，不会执行真实逻辑，而是返回 "mocked data"。

  3. Verify（验证）：

    • 验证某个方法是否被调用，以及被调用了多少次。
    • 例如：verify(mockService, times(1)).saveData(anyString());
    • 这行代码会检查 saveData 方法是否被调用了恰好一次。

  4. Spy（部分模拟）：

    • 对真实对象进行部分模拟。（只模拟自己写的业务，三分框架的业务继续正常调用）
    • 默认情况下，调用真实的方法；但你可以对某些方法进行 stub，覆盖其行为。
    • 与 @Mock 不同，@Spy 创建的是真实对象的包装。
  • @Mock + @InjectMocks：不启动 Spring 容器，完全手动控制依赖，速度快，适合纯单元测试。
  • @MockBean：启动 Spring 容器，但将某个 Bean 替换为 Mock，适合集成测试中隔离外部依赖。

Mock+@InjectMocks

纯单元测试，测试InjectMocks的逻辑，其他的模拟，不启动Spring

使用 @SpringBootTest + @MockBean

部分启动Spring容器但是替换某个Bean为Mock

使用 @SpyBean

测试真实对象，模拟其中部分方法

MockMVC

必须在Spring测试上下问中使用，是Springtest框架的一部分，上面的Mockito是独立的框架，可以单独使用

  1. 创建模拟的MVC环境，模拟发送请求，而不是真实的HTTP客户端

  2. 请求不会真正通过网络传输，而是在Spring MVC框架内部直接调用Controller

  3. mockMvc.perform()方法模拟HTTP请求处理流程

  4. 启动Springboot项目，只是以测试模式运行而非完整的Web服务器

  5. 通常与Mock搭配使用

使用 @WebMvcTest（可选）： 如果你想让 Spring 容器管理 Controller 的依赖注入（而不是手动 @InjectMocks），可以使用：

接口MockMvcBuilder，提供一个唯一的build方法，用来构造MockMvc。主要有两个实现：

  • StandaloneMockMvcBuilder 独立安装测试

  • DefaultMockMvcBuilder 集成Web测试，无需启动真实服务器

MockMvcBuilders提供了对应的创建方法standaloneSetup方法和webAppContextSetup方法，在使用时直接调用即可。

  1. mockMvc.perform执行一个请求
  2. MockMvcRequestBuilders.get(“”)构造一个请求
  3. ResultActions.param添加请求传值
  4. ResultActions.accept(MediaType.TEXT_HTML_VALUE))设置返回类型
  5. ResultActions.andExpect添加执行完成后的断言。
  6. ResultActions.andDo添加一个结果处理器，此处使用MockMvcResultHandlers.print()输出整个响应结果信息。
  7. ResultActions.andReturn表示执行完成后返回相应的结果
